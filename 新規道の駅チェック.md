# 新規道の駅チェックの仕組み(有効化は保留中)

## 内容
週1回、国土交通省の道の駅一覧(https://www.mlit.go.jp/road/Michi-no-Eki/list.html の Excel)を確認し、
`data.js` に載っていない新しい道の駅(プレオープン含む)があれば追加し、ユーザーに知らせる。

## 有効化したいとき
ユーザーが「新規道の駅チェックを有効にして」と言ったら、CronCreateで以下のプロンプトを
週1回(例: 毎週月曜 9:03)実行するジョブを作成する。
※CronCreateの自動ジョブは7日で失効するため、有効化時はその旨を伝え、必要なら再設定する。

## 実行プロンプト(チェック処理の内容)
1. https://www.mlit.go.jp/road/Michi-no-Eki/file/list.xls をダウンロード
2. Excelで開いてCSV変換し、`県名`+`駅 名`の組み合わせを
   `C:\Users\81906\OneDrive\デスクトップ\michinoeki_app\data.js` の MICHINOEKI_DATA と比較
3. 新規に増えている道の駅があれば、`data.js` に以下を追加する
   - `id`: 既存の最大id+1から連番
   - `pref`, `name`, `round`, `date`, `location`, `url`: 一覧から取得
   - `isNew`: true
   - `addedAt`: チェックした日付(YYYY-MM-DD)
4. 新規追加があった場合、ユーザーに「新しい道の駅が追加されました: ○○県 ○○」と通知する
   (PushNotificationツールなどを使う)
5. 新規追加が無かった場合は通知しない(静かに終了)

## 既存データの「isNew」について
- アプリ側では `isNew: true` の駅に NEW バッジが表示され、上部バナーにも一覧表示される
- バナーの「OK」ボタンを押すと、その駅は今後バナーに出なくなる(NEWバッジは残る)
- `isNew` を完全に消したい場合は、`data.js` の該当エントリから `isNew` フィールドを削除する
